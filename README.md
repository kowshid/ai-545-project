# Insurance Charge Predictor

End-to-end MLOps demo: trains a regression model on the
[Medical Cost Personal Dataset](https://www.kaggle.com/datasets/mirichoi0218/insurance)
and serves predictions through a Streamlit app.

**Stack:** scikit-learn · MLflow (experiment tracking) · file-based model
registry · Docker · Kubernetes (minikube) · GitHub Actions · GHCR · Hugging
Face Spaces (public URL) · New Relic (optional APM).

The top YAML block above is consumed by Hugging Face Spaces to build the
Space as a Docker container.

## 2. Production flow

```
git push main
     │
     ▼
GitHub Actions (ci-cd.yml)
 1. train          → MLflow run + model.pkl + metrics.json      (artifact)
 2. register       → compare to current, write champion.json     (artifact)
 3. build-image    → docker build, push to ghcr.io/<user>/insurance-app
 4. deploy-hf      → git push to huggingface.co/spaces/<user>/insurance-charge-predictor
                          │
                          ▼
                    HF Space rebuilds container → public URL live
```

- **Public URL:** `https://<hf-user>-insurance-charge-predictor.hf.space`
- **Container image:** `ghcr.io/<github-user>/insurance-app:latest` — used by Kubernetes
- **MLflow runs + registry:** downloadable as workflow artifacts in the Actions tab

## 3. One-time setup

### Prerequisites

| Tool               | macOS                           | Windows                                                 |
|--------------------|---------------------------------|---------------------------------------------------------|
| Python 3.11        | `brew install python@3.11`      | [python.org](https://www.python.org/downloads/)         |
| Docker             | Docker Desktop                  | Docker Desktop                                          |
| kubectl + minikube | `brew install kubectl minikube` | `winget install Kubernetes.kubectl Kubernetes.minikube` |
| GitHub CLI         | `brew install gh`               | `winget install GitHub.cli`                             |

### Create the Hugging Face Space

1. Go to <https://huggingface.co/new-space>
2. Name: `insurance-charge-predictor`
3. SDK: **Docker** (*not* Streamlit — that SDK is deprecated)
4. Visibility: Public

### GitHub secrets

Get a write-access HF token from <https://huggingface.co/settings/tokens> first, then:

```bash
gh secret set HF_TOKEN --body "hf_xxxxxxxxxxxxxxxx"
gh secret set HF_USER  --body "<your-hf-username>"
```

### Make the GHCR image public (so k8s can pull it without auth)

After the first successful CI run, go to:
**GitHub profile → Packages → insurance-app → Package settings → Change visibility → Public**


## 5. Kubernetes (minikube)

The same GHCR image that CI builds is what k8s runs.

```bash
minikube start
# (If image is public, k8s can pull it directly. If it's private,
#  create a pull secret — see Kubernetes docs on imagePullSecrets.)

kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
kubectl rollout status deployment/insurance-app-deployment
minikube service insurance-app-service --url

# Scale out
kubectl scale deployment insurance-app-deployment --replicas=4

# Rolling update — happens automatically when GHCR gets a new :latest tag
#   and a pod restarts, or force one with:
kubectl rollout restart deployment/insurance-app-deployment
```

Clean up:

```bash
kubectl delete -f k8s/service.yaml -f k8s/deployment.yaml
minikube stop
```

## 6. Day-to-day loop

```bash
# edit code
git commit -am "..."
git push origin main

gh run watch      # live CI status
```

Every push re-trains the model, compares to the current champion, and — if
the new model wins on R² — bakes it into the image and redeploys the HF Space.

## 7. New Relic (optional)

1. Get an ingest license key from New Relic (40 chars, ends in `NRAL`).
2. Add it to the HF Space as a secret:
   HF Space → **Settings → Variables and secrets → New secret**
    - Name: `NEW_RELIC_LICENSE_KEY`
    - Value: your key
3. Restart the Space. `entrypoint.sh` detects the env var and starts
   Streamlit wrapped in `newrelic-admin run-program`.

For Kubernetes, use the template in `k8s/newrelic-secret.example.yaml`.

## 8. Troubleshooting

- **HF Space shows "Build failed".** Open the Space → **Logs** tab. Usually
  a missing file or a too-large `model.pkl` (HF's free tier has a 10 GB
  total limit per Space, individual files should stay under ~50 MB).
- **k8s pod in `ImagePullBackOff`.** The GHCR package is still private —
  flip it to public or create an `imagePullSecret`.
- **GitHub Actions: Node 20 deprecation warning.** Already handled by
  `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: true` in the workflow `env:` block.
- **Champion never updates.** Expected — `src/register.py` only promotes
  when the new R² is strictly greater. Edit the comparison there if you
  want different promotion logic.
